Introduction

Sorry for my bad English :) I hope you'll be able to understand this manual. If you have any questions, ask at the RHDN topic.

BZK 6502 Disassembler is a Lua script that is heavily and solely based on a .cdl file. What the script does is separates code from data, it doesn't make any assumptions like some other disassemblers. The more complete CDL file you possess, the more information you will get and the easier for you will be to create an actual assembly of the game.

The final goal is to make output files compatible with a ca65 assembler. All scripts will save you at least 50% of your time. However, creating an actual working disassembly of the game, where you can edit and move any code you want, is still a pain in the ass even with my scripts, and not many romhackers are willing to go through it.

That's why the very basic goal of my disassembler it simply to have a better view at the game code using a text editor, instead of looking at it via Debugger, even if you don't have much romhacking skills. I tried to make everything as simple as I could afford.

Step 1 is to make a CDL file, edit config and launch the disassembler to create assembly files. Step 2 is to polish this files with Notepad++ script. It will take you a couple of hours in total. Of course if you want to create a perfect disassembly for editing the code in a game, you need to spend much more time on it and then go to step 3 - prepare these assembly files for an assembler.

Features

Disassembler


  • Separate code from data using a CDL file
  • Display all 6 CDL flags
  • Display ROM File and NES Memory address
  • Use several table files to display symbols
  • Auto adding labels to RAM addresses
  • Additional info about each CPU address

    • Notepad++ script


  • Convert .byte to instructions
  • Convert .byte to .word/.dbyt + labels
  • Auto adding labels to Branches, Tables and JMP/JSR instructions
  • Other manipulations with bytes

    • And a couple of bonuses:

  • Syntax highlighting
  • Display matches counter of a selected text

    • Preparation before ca65 compilation (will be completed soon)

    The idea is to have your assembly files as is (like keeping flags and addresses in files), but still being compatible with an assembler by launching one more script each time before compilation.

  • Make a basic compatibility with ca65
  • Allow user to write labels in any language
  • Convert text from any language into a list of bytes
  • Create .ram.nl file for FCEUX
