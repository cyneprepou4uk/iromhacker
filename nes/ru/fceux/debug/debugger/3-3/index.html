Типы брейкпоинтов

Тип указывается в окне создания/редактирования брейкпоинта.

Брейкпоинт можно создать и без указания типа. В любой момент его можно отредактировать.

Типы Read (чтение), Write (запись) и Execute (выполнение) могут быть активны одновременно. Тип Forbit (запрет) выставляется отдельно и блокирует все остальные типы.

Список команд, активирующих Read и Write брейкпоинты, смотри в разделе с командами процессора.

Причина кроется в подсветке брейкпоинта в списке, условие которого было выполнено. Если создать 2 отдельных брейкпоинта с одинаковым условием на адрес, и разделив между ними условия Read и Write, то при срабатывании брейкпоинта ты сразу поймешь какой из них сработал по синему выделению.

Но если ты создашь один брейкпоинт с условиями Read и Write одновременно, он будет подсвечен независимо от того, какое из этих двух условий было выполнено.

Понять это можно будет только по команде, которая вызвала срабатывание брейкпоинта. Поэтому лучше не выставлять эти 2 условия в одном брейкпоинте одновременно.

Read брейкпоинты

Условие на чтение из адреса. Брейкпоинт сработает при попытке процессора считать байт из адреса, указанного в условии брейкпоинта. Если что-то считывается из адреса, нужно ставить условие Read.

С помощью условия Read можно понять для какой цели служит некий адрес, определив код, который считывает его байт.

Read брейкпоинт отображается с флагом R.

Write брейкпоинты

Условие на запись в адрес. Брейкпоинт сработает при попытке процессора записать байт в адрес, указанный в условии брейкпоинта. Если что-то записывается в адрес, нужно ставить условие Write.

С помощью условия Write можно найти, например, запись начального количества жизней, здоровья, таймеров.

Write брейкпоинт отображается с флагом W.

Execute брейкпоинты

Условие на выполнение команды по адресу. Брейкпоинт сработает при попытке процессора выполнить команду, расположенную по адресу, указанному в условии брейкпоинта.

С помощью условия Execute можно определить предназначение незнакомой подпрограммы, определив игровой момент, в котором ее код будет использован.

Тип Execute не доступен для адресов PPU Memory и Sprite Memory.

Чтобы быстро добавить Execute брейкпоинт по нужному адресу, достаточно дважды кликнуть по адресу слева от команды и сохранить предложенный брейкпоинт.

Execute брейкпоинт отображается с флагом X.

Forbit брейкпоинты

Противоположность Execute. Запрещает срабатывать остальным брейкпоинтам при выполнении команды по адресу, указанному в условии.

С помощью условия Forbit можно отключить лишние срабатывания брейкпоинтов в определенных местах. При проверке Debugger'ом списка брейкпоинтов на срабатывание, у Forbit приоритет выше, чем у остальных типов.

Можно быстро добавить Forbit брейкпоинт точно также, как и Execute, только выставив условие Forbit перед сохранением.

Forbit брейкпоинт отображается с флагом F.
