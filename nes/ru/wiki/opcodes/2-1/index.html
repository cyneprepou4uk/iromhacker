Дополнительные режимы адресации

Indexed

Индексная адресация. При вычислении адреса учитывается значение индексного регистра X или Y. К адресу, указанному в операнде команды, добавляется текущее значение индексного регистра во время выполнения команды. После сложения операнда и индексного получается итоговый адрес, с которым будет работать команда.

Команда, работающая с индексным регистром, не может использовать этот же регистр для режима индексной адресации. То есть не существует команд вроде LDX,X или LDY,Y.

Поскольку значение индексного регистра может быть в диапазоне от #$00 до #$FF, при помощи одной-единственной команды можно обратиться к целому диапазону размером 256 байтов, изменяя значение индексного регистра перед выполнением этой команды.

Этот подход очень часто применяется при создании циклов и подпрограмм. А также индексная адресация - одна из наиболее важных причин, почему однотипные адреса у различных игровых объектов, например их координаты, очень часто находятся в RAM по соседству.

Индексная адресация может быть использована как с режимом адресации Zero Page, так и Absolute. У некоторых команд есть ограничение по выбору этих разновидностей индексной адресации, в основном ограничения по использованию регистра Y.

Также индексная адресация может использоваться с режимом адресации Indirect, которая описана ниже.

Zero Page Indexed

Поскольку указывается адрес нулевой страницы, операн команды должен быть размером 1 байт.

00:8100: A9 10 LDA #$10 ; загрузить байт #$10 в регистр A
00:8102: A2 00 LDX #$00 ; загрузить байт #$00 в регистр X
00:8104: 95 99 STA $99,X ; записать байт из регистра A в адрес $0099
00:8106: E8 INX ; увеличить байт в регистре X на #$01
00:8107: 95 99 STA $99,X ; записать байт из регистра A в адрес $009A
00:8109: E8 INX ; увеличить байт в регистре X на #$01
00:810A: 95 99 STA $99,X ; записать байт из регистра A в адрес $009B

Чтобы команда STA использовала режим адресации Zero Page Indexed, она записывается с помощью опкода 95. А чтобы записать байт из регистра A в адрес $0099,X, после опкода ставится операнд 99.

Изменяя байт в регистре X, изменится и итоговый адрес, в который будет записан байт из регистра A.

Нельзя выйти за пределы нулевой страницы, используя адресацию Zero Page Indexed. Например, выполняя команду STA $FF,X с текущим значением X = #$01, байт из регистра A будет записан в адрес $0000, а не $0100.
